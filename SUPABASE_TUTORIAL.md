Follow these steps to create the necessary tables for the **Daily Seongsu** project.

### Step 1: Login to Supabase
1. Go to [Supabase Dashboard](https://supabase.com/dashboard/projects).
2. Select your **Daily Seongsu** project.

### Step 2: Open SQL Editor
1. In the left-hand sidebar, look for the **SQL Editor** icon (it looks like a `>_` symbol).
2. Click on it.
3. Click on **"+ New query"** (or use the default "New Query" tab).

### Step 3: Copy and Paste the SQL
Copy the following SQL code and paste it into the editor:

```sql
-- 1. Create table for daily subway traffic
create table if not exists subway_traffic (
  id bigint generated by default as identity primary key,
  date date not null,
  station_name text not null,
  line_number text not null,
  boarding_count int,
  alighting_count int,
  created_at timestamp with time zone default timezone('utc'::text, now()),
  
  -- Prevent duplicate entries for the same date/station/line
  unique(date, station_name, line_number)
);

-- 2. Create table for real-time weather data
create table if not exists weather_data (
  id bigint generated by default as identity primary key,
  measured_at timestamp with time zone not null,
  temperature float,
  precipitation_type int,
  humidity float,
  created_at timestamp with time zone default timezone('utc'::text, now())
);
```

### Step 4: Run the Query
1. Click the **"Run"** button in the bottom right corner of the SQL Editor.
2. You should see a message saying **"Success. No rows returned"**.

### Step 5: Verify
1. Click on the **Table Editor** icon (looks like a table/grid) in the left-hand sidebar.
2. You should now see two new tables: `subway_traffic` and `weather_data`.
